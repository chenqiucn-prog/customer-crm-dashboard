销售账号 sales

只能查看、编辑、删除 owner_user_id 等于自己的项目；新增项目自动归属自己。

总经理 general_manager

查看全量项目，可按负责人筛选、可新建项目并指定归属销售。

管理员 admin

拥有全量项目和配置管理权限，用于系统维护。

后端权限策略

不是单纯前端隐藏

Supabase Auth

前端邮箱密码登录，获取 access token。

Netlify Functions

每次 API 请求校验 token，并读取 app_users 角色。

PostgreSQL RLS

数据库层也设置 owner_user_id 与角色策略，形成双层防护。

新增销售项目

请按“归属负责人 → 客户信息 → 产品与金额 → 推进状态 → 应用场景与下一步动作”的顺序填写。带 * 的字段建议必填。

一、项目归属

销售账号新增项目时自动归属本人；总经理/管理员可以指定项目归属销售。

二、客户信息

用于识别客户单位、联系人和客户价值等级。

三、产品与金额

先选产品线/解决方案，再选择该产品线下的具体产品名。

四、项目推进状态

用于销售漏斗、重点项目排序和风险预警。

五、应用场景与下一步动作

建议写清楚客户为什么需要、下一步由谁做、什么时候完成。
